# Auto-link LLC Website (New Version)

A clean, static dealership site. No database, no admin page — your inventory lives in `Cars/` folders and is rendered from `js/cars-data.js`.

## How inventory works

1. In the project's `Cars/` folder (next to this `New version website` folder), make a folder per car. Name it however you like — e.g. `2017 Hyundai Sonata`.
2. Drop the car's photos (JPG/PNG) directly into that folder.
3. Add a plain-text `Info.txt` file with three lines:

   ```
   VIN: 5NPE34AF6HH529183
   Mileage: 112833
   Price: $9500
   ```

4. Ask the agent to "update the website" — it will:
   - Decode the VIN via the NHTSA vPIC API.
   - Use the VIN's year/make/model/submodel as the source of truth (overrides folder name if different).
   - If the folder has photos, copy them into `img/cars/<slug>/`.
   - If the folder has **no photos**, the car still appears with a *"Photos coming soon"* placeholder.
   - If the folder has **no `Info.txt`**, the car still appears with *"Information coming soon"*.
   - Append/replace the car entry in `js/cars-data.js`.

## What was removed vs. the old site

- `admin.html` and all admin auth/JS
- `js/cars-jsonbin.js`, `js/cars-firebase.js`, `js/firebase-config.js`
- JSONBin.io and Firebase setup/connection — no third-party DB anymore
- All JSONBin/Firebase/database setup `.md` docs

## What is still external

- **Formspree** (`https://formspree.io`) still handles the *financing* and *trade-in* form submissions. The Content-Security-Policy in each page allows only `formspree.io` for outbound requests.

## Deploy to Netlify (automatic)

This project is set up for one-command deploys:

```powershell
npm run deploy -- "Mark 2016 Equinox as sold"
```

That commits any changes, pushes to GitHub, and deploys to Netlify (if credentials are configured).

### One-time setup

**Option A — GitHub auto-deploy (recommended)**

1. In [Netlify](https://app.netlify.com/), open your site (`autolinkllc-co`).
2. Go to **Site configuration → Build & deploy → Continuous deployment**.
3. Link the GitHub repo [DirtyRaandy/autolinkllc-website](https://github.com/DirtyRaandy/autolinkllc-website).
4. Build command: leave empty. Publish directory: `.` (root).

After that, every `npm run deploy` push updates the live site automatically.

**Option B — Netlify CLI (no GitHub link needed)**

1. Copy `.env.example` to `.env`.
2. Create a [Netlify personal access token](https://app.netlify.com/user/applications#personal-access-tokens).
3. Copy your **Site ID** from Netlify → Site configuration → Site details.
4. Paste both values into `.env`.

Future agent sessions can run `npm run deploy` after website changes.
